Perfect use of Inkscape with Latex

Posted on 28/10/2010 by Benjamin

Making figures for Latex documents can be a pain, as you first have to draw the figures, then cut the figures and export to .pdf or .eps, before inserting into the document. In addition, a small change in a figure or a text in a figure makes you have to do it all again. Also, when using version control of the documents, you must remember to add every single figure drawn, possibly both the Inkscape .svg file as well as the .eps or .pdf.

However, I discovered two things making the above very easy and elegant. The first is the whole cutting and exporting to .eps as well as adding to svn. Making a small bash script will do the job for you, including converting the figures to the desired format, as well as adding to svn. In this way you never have to worry about converting files and forgetting to svn add.

To make it all very neat, you can conveniently add a shortcut in your favorite tex editor running the bash script. All of this is described in the following along with a brief intro to the infamous Latex package psfrag, making it possible to write the figure texts directly in Latex code.

The following example script shows the idea of the .svg to .esp (or .pdf) conversion.

view source
print?
01#!/bin/bash
02 
03#the folder containing your .svg files
04cd /home/biegel/documents/svn/figures/svg
05 
06for picture in *.svg
07do
08echo $picture
09pictureChanged=$(stat -c %Y $picture)
10currentTime=$(date +%s)
11pictureChanged=$(stat -c %Y $picture)
12currentTime=$(date +%s)
13echo Picture changed: $pictureChanged
14echo Current time $currentTime
15echo Time picture became old: $((pictureChanged+3600))
16if  [ $((currentTime)) -gt $((pictureChanged+3600)) ];
17then
18    echo  $picture not recently edited
19else
20    echo converting $picture '->' ${picture/.svg/}.eps
21    inkscape $picture --export-eps=${picture/.svg/}.eps
22    # Move the .eps file to your eps folder
23    mv ${picture/.svg/}.eps ../eps/.
24    # Optional: Add both .svg and .eps to svn
25    svn add $picture
26    svn add ../eps/${picture/.svg/}.eps
27fi
28 
29done
30exit 0

Now running the above script will do the job, converting your inkscape .svg files to .esp files. The script optionally looks at the time the .svg file last was edited, and only converts the file if this is less than 1 hour ago. This can come in handy if you have a lot of figures.

Adding the script as a shortcut to your text editor is very convenient. Doing so in Kile is simple: Go to Settings -> Configure Kile -> Build and choose ‘New…’ and create a configuration tool called something like ‘svg convert’ and in the ‘Command’ field enter the path to the file above. Choose ‘Run Outside of Kile’ in the Advanced settings. After this save and exit the build menu and go to Settings -> Configure Shortcuts. Search for svg (if that is what you called the built command you just made), and enter your favourite shortcut. Now, simply pressing this shortcut, will run the script.

If you often edit figures, you can even put this command into your Quick Build option (also found in the Build settings).

Last, but not least, the Latex package Psfrag lets you enter text on figures directly into Latex code, rendering it possible to change the texts easily. If you are using the newest version of Inkscape, there is however an error in the software, so you need to a small amount of work to fix this problem. In post #47 here, Adrian Johnson explains how to solve the problem. After following that step, you simply replace the line in the above bash code where it says:

view source
print?
1inkscape $picture --export-eps=${picture/.svg/}.eps

with:

view source
print?
1LD_LIBRARY_PATH=$HOME/cairo-git/lib inkscape $picture --export-eps=${picture/.svg/}.eps

and you are all set!
